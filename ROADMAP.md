# WrenchPro Roadmap

## Roadmap Philosophy

WrenchPro should first become dependable for daily business use before chasing big new features.

Priority order:

1. Reliability
2. Core business workflow
3. Data safety
4. Usability
5. Automation / advanced features

## Phase 1 — Stabilize Current App

Goal: make the current feature set safe and usable.

- [ ] Identify all existing features/screens.
- [ ] Complete manual QA pass.
- [ ] Log bugs in `KNOWN_ISSUES.md`.
- [ ] Fix release-blocking bugs.
- [ ] Confirm database backup behavior.
- [ ] Confirm installer/update behavior.
- [ ] Confirm no broken navigation/buttons.

## Phase 2 — Fit the Real Business Workflow

Goal: match how the mobile mechanic actually works.

- [ ] Map job lifecycle from first contact to payment/follow-up.
- [ ] Confirm required customer fields.
- [ ] Confirm required vehicle fields.
- [ ] Confirm estimate/invoice requirements.
- [ ] Confirm parts/inventory needs.
- [ ] Confirm warranty and follow-up needs.
- [ ] Confirm reports needed for taxes/business tracking.

## Phase 3 — Release Candidate

Goal: produce a trustworthy build for real use.

- [ ] Resolve all release blockers.
- [ ] Run release checklist.
- [ ] Build installer.
- [ ] Test install on local machine.
- [ ] Verify app launches from shortcut.
- [ ] Verify database persists after update/reinstall.
- [ ] Draft release notes.
- [ ] Brandon approves release.

## Phase 4 — Business Enhancements

Possible future improvements:

- customer reminders
- service history reports
- recurring maintenance reminders
- parts catalog improvements
- quote-to-job conversion
- invoice/payment tracking
- mileage/travel tracking
- profit per job
- technician time tracking
- simple mobile-friendly view
- cloud sync / multi-device support

## Parking Lot

Ideas that may be useful later but should not distract from stabilization:

- full accounting integration
- online booking
- SMS/email automation
- customer portal
- payment processor integration
- inventory barcode scanning
- multi-employee scheduling
