# WrenchPro Support and Documentation Plan

As WrenchPro becomes a real software product, support and documentation need to grow with it.

## Support Philosophy

- Be practical and clear.
- Help users get back to work quickly.
- Treat bugs seriously.
- Separate software issues from training issues.
- Keep a record of repeated questions and pain points.

## Early Support Channels

For now, support should stay controlled:

- Brandon / direct conversations
- brother's feedback
- private tester feedback
- GitHub issues only if Brandon wants that public later

Do not open public support channels until the product is ready.
